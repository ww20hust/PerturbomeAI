Methods

Simulation framework for phenotype-to-perturbation inference

We first used a controlled simulation to define the statistical conditions under which genetic perturbations can be recovered from phenotypic measurements. The simulation was designed as a forward genotype-to-phenotype system with known ground truth, so that the reverse inference problem could be evaluated without ambiguity. Twenty independent biallelic loci were simulated in a cohort of fifty thousand individuals. Locus frequencies were distributed from rare to common, and the effect assigned to each locus was larger for rare alleles and smaller for common alleles, reflecting the expected action of negative selection on deleterious or functionally consequential variation. Genotypes were sampled independently for each participant, and two continuous phenotypic traits were generated by combining locus-specific effect directions with Gaussian environmental and stochastic variation. This design produced a transparent phenotype space in which each locus had a known direction and magnitude of effect, while all loci contributed jointly to background phenotypic variation.

Carrier status was then inferred from the simulated phenotypes using the same reverse modeling logic used in PerturbomeAI. For each simulated locus, individuals carrying at least one alternate allele were treated as carriers and all remaining individuals were treated as non-carriers. XGBoost classifiers were trained from the two simulated phenotypes and derived geometric features, and performance was evaluated by eight-fold cross-validation. The simulation was used to examine three related questions: whether carrier status could be recovered from a multivariate phenotype, how recoverability changed across the allele-frequency spectrum, and whether frequency or effect size was the dominant driver of model performance. To separate these factors, additional cohorts were generated in which either effect size was held fixed while frequency varied, or frequency was held fixed while effect size varied. Training-set downsampling was then performed for representative rare and common loci to assess how carrier representation affected recoverability. These experiments established a controlled benchmark for interpreting real biobank results and for distinguishing signal-limited models from sample-size-limited models.

Locus-specific PerturbomeAI model training and GPS quantification

PerturbomeAI models each genetic locus as a separate phenotype-to-perturbation task. For a given single-nucleotide variant, pharmacogenomic variant, regulatory variant, or gene-level collapsed variant set, genotype data were converted into binary labels. Individuals carrying the target perturbation were assigned positive labels, and non-carriers were assigned negative labels. The model input was the participant-level phenotypic feature matrix after preprocessing and missing-value completion. A separate XGBoost model was trained for each locus so that the learned score represented the phenotypic imprint of that locus rather than a shared multi-locus prediction target.

Cross-validation used an eight-fold design. For each rotation, five folds were used for training, two folds were used for validation and early stopping, and one fold was held out for testing. The held-out fold was rotated across all eight folds, ensuring that every participant received exactly one out-of-fold score from a model that had not been trained on that participant. Class imbalance was handled during training by weighting the positive class according to the non-carrier to carrier ratio estimated within the training folds, and the same imbalance-aware weighting was applied to the validation data used for early stopping. This design was used to reduce overfitting, preserve a strict out-of-fold evaluation, and provide a continuous score for every individual in the cohort.

The raw out-of-fold model outputs were aligned across folds before downstream analysis. Because each test fold is scored by a different trained model, score distributions can differ slightly across folds even when within-fold ranking is meaningful. To make scores comparable across the full cohort, each fold-specific score distribution was mapped to a reference fold by empirical rank percentile. This procedure preserves the ordering of individuals within each fold while placing all out-of-fold scores onto a common scale. The resulting aligned value was defined as the locus-specific Genetic Perturbation Score, or GPS.

Model discriminability was quantified by comparing aligned GPS distributions between factual carriers and non-carriers. Area under the receiver operating characteristic curve was used to measure rank-based separation. Cohen's d was used to summarize standardized separation between the two distributions. Cliff's delta and the Mann-Whitney test were used as non-parametric measures of distributional shift. Median GPS difference between carriers and non-carriers was also calculated as an interpretable measure of effect direction and magnitude. For analyses involving many loci, p-values were adjusted for multiple testing using the Benjamini-Hochberg false discovery rate procedure.

Genome-scale parallel computation for hundreds of thousands of loci

Genome-scale GPS modeling requires training and scoring a very large number of independent locus-specific models. Although each locus is statistically independent after labels are defined, all loci share the same participant-level phenotype matrix and the same eight-fold cohort partition. PerturbomeAI therefore uses a block-based parallel computation strategy that separates shared data access from locus-specific model fitting. The cohort is assigned to eight cross-validation folds once, and that assignment is reused for every locus. The preprocessed feature matrix is written once as a read-only shared memory-mapped array, so training jobs do not repeatedly parse or copy the same phenotype data. Locus labels are partitioned into blocks, and each block stores the binary labels for many loci together with a block manifest.

The smallest training unit is one block and one test fold. In this unit, the job loads the shared feature matrix and the block label matrix, selects the five training folds, two validation folds, and one test fold, and then trains one weighted XGBoost model per locus in the block for that fold. The eight folds for a block are scheduled together, allowing the eight fold models to map naturally onto eight GPUs. Because models for different loci do not exchange parameters, many blocks can be processed in waves, with jobs dispatched round-robin across available devices. While one shard is being scored and merged, the next shard can begin training, which overlaps post-processing with model development and reduces idle time. After all fold jobs complete, held-out predictions are assembled into a full out-of-fold score matrix, fold-wise percentile alignment is applied, and per-locus discrimination statistics are computed. This design exploits the independence of locus-specific models while minimizing repeated feature I/O, enabling hundreds of thousands of loci to be developed and cross-scored on a practical time scale.

Phenotype preprocessing and missing feature completion

The primary phenotypic input consisted of sixty-two routine clinical variables available in population-scale biobank data. These variables included complete blood count measurements, blood biochemistry, urine assays, body mass index, and age. Because the measurements span different biological systems and units, each non-age feature was normalized column by column using a quantile-based transformation fitted on observed values. Age was scaled separately. Missing values were retained during normalization and then completed using a masked beta variational autoencoder. The autoencoder was trained by hiding a random subset of observed entries and reconstructing them from the remaining measurements, which teaches the model to infer missing clinical indicators from the observed physiological context. At inference, observed measurements were retained and missing entries were replaced by reconstructed values, producing a complete feature matrix for GPS modeling.

To assess whether the PerturbomeAI framework could extend beyond routine laboratory tests, we also considered retinal optical coherence tomography as an alternative phenotypic modality. For this analysis, we used derived quantitative retinal features available in the UK Biobank under Category 100079, which contains quantitative measurements extracted from retinal optical coherence tomography images. These features were treated as participant-level phenotypic inputs under the same general framework: phenotypes were normalized, locus-specific carrier labels were defined from genotype data, and the resulting feature matrix was used to train perturbation-specific GPS models. This analysis tested whether the framework could generalize from routine clinical assays to structured imaging-derived phenotypes.

GPS-GWAS

GPS-GWAS treated each aligned locus-specific GPS as a quantitative trait. After standard genotype quality control, genome-wide association testing was performed with baseline age, sex, and forty exome-derived principal components as covariates. The principal components were computed with PLINK2 from exome data. Genome-wide association analyses were carried out chromosome by chromosome using the PLINK2 additive linear model, with covariates variance-standardized before regression. For the APOL1 locus, an additional subgroup GWAS was performed among carriers of established APOL1 risk variants to search for protective alleles that might modify the GPS-defined perturbation state. Genome-wide significance was defined using a threshold of one times ten to the minus eight. Variant annotation was performed using GENCODE version forty-four.

GPS-proteomics associations

Plasma proteomic measurements were obtained from the UK Biobank Pharma Proteomics Project, which profiled about two thousand nine hundred and twenty-three proteins in about fifty thousand participants using the Olink Explore platform. For each locus-specific GPS, aligned scores were merged with available proteomic data by participant identifier. The GPS was transformed to a percentile-based Z-score so that regression coefficients were comparable across loci and robust to differences in score distribution. Each protein was then modeled as a continuous outcome using ordinary least-squares regression with the transformed GPS as the primary explanatory variable, adjusting for age and sex. Multiple testing was controlled within each omics panel using the Benjamini-Hochberg false discovery rate procedure. This analysis identified proteins whose circulating abundance tracked the continuous perturbation state captured by the GPS, enabling molecular characterization even when factual carriers were rare.

Longitudinal stability

Longitudinal stability was assessed by comparing GPS values between baseline assessment and the first repeat assessment visit. Participants were included when both visits were available and the interval between visits exceeded five years. After inner-joining the paired participants, scores from each visit were converted to their own rank-percentile scale to remove visit-specific distributional shifts. Spearman correlation coefficients were then computed across the full paired cohort. This analysis evaluated whether a locus-specific GPS captured a stable individual physiological state rather than a transient measurement artifact. The same paired-visit logic was applied in the CHAI cohort for participants who met the minimum five-year inter-visit interval.

SHAP and ablation analyses

Model interpretation was performed using Shapley additive explanations computed from XGBoost models trained on the full labeled baseline set with the same feature space used for GPS modeling. For each gene-specific model, SHAP values were summarized globally to identify the phenotypic measurements most strongly contributing to the predicted perturbation score. These analyses provided an interpretable view of the clinical systems represented in each locus-specific signature.

Feature ablation was then used to test whether model signal depended on a single dominant downstream biomarker or was distributed across the phenotypic panel. The top predictors identified from model attribution were removed individually and jointly from the input matrix, models were retrained using the same cross-validation and alignment procedure, and discrimination metrics were recomputed for each ablated feature set. Loss of performance after removing a small number of predictors suggested concentrated signal, whereas stable performance after ablation supported a distributed multi-feature signature.
